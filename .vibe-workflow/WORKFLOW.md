# AI Coding 阶段闸门工作流

## 总流程

```text
需求采集/澄清
  ↓
需求设计
  ↓
详细设计
  ↓
前端详细设计 + 后端详细设计
  ↓
代码编写
  ↓
前后端单元测试
  ↓
集成测试
  ↓
Playwright E2E
  ↓
Review
  ↓
知识沉淀
```

## 每个阶段的通过条件

| 阶段 | 通过条件 |
|---|---|
| 需求澄清 | 目标、非目标、验收标准明确 |
| 需求设计 | 用户旅程、业务规则、状态流转明确 |
| 详细设计 | 模块、数据流、接口、风险、回滚明确 |
| 前端设计 | 页面、组件、状态、请求、测试路径明确 |
| 后端设计 | API、数据模型、权限、事务、错误处理明确 |
| 代码编写 | 一个最小切片实现完成，未越界修改 |
| 单元测试 | 核心规则、边界、异常路径覆盖 |
| 集成测试 | 模块协作、权限、数据读写验证 |
| E2E | 用户关键路径浏览器验证通过 |
| 知识沉淀 | 关键设计、踩坑、改进建议已记录 |
