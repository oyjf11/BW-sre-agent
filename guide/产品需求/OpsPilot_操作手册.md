## 1. 产品简介

### 1.1 什么是 OpsPilot

OpsPilot 是您的**智能运维故障处置助手**。它可以帮您：

- **自动分析故障**：收到工单或告警后，自动生成排查计划
- **收集证据**：并行查询日志、监控指标、发布记录、知识库
- **给出建议**：输出可能的根因及修复方案，包含风险等级
- **执行动作**：低风险操作自动执行，高风险操作需您审批
- **生成报告**：处置完成后自动生成复盘报告（RCA）

### 1.2 核心价值

| 能力 | 带来的好处 |
|------|------------|
| 自动化排查 | 无需手动翻找日志和监控，AI 帮您收集证据 |
| 结构化分析 | 给出根因候选及置信度，不是"蒙答案" |
| 风险分级 | 低风险自动处理，高风险人工审批，安全可控 |
| 知识沉淀 | 处置经验自动沉淀到知识库，下次类似问题更快解决 |

### 1.3 典型使用场景

- **告警触发**：支付服务 5xx 飙升、接口响应变慢
- **工单投诉**：用户反馈订单失败、页面无法访问
- **发布后异常**：新版本上线后出现错误
- **周期性问题**：凌晨定时任务失败、批处理堆积

## 2. 快速开始

只需三步完成一次故障处置：

### 步骤 1：提交工单

在工单输入页面填写：

- **工单描述**（必填）：粘贴告警内容或故障描述
- **服务名**（建议填写）：如 `payment-service`、`order-api`
- **环境**：生产（prod）/预发（staging）/测试（test）
- **时间范围**：故障发生的时间段

点击"开始处理"，OpsPilot 随即启动。

### 步骤 2：等待分析与审批

系统会自动完成：

1. **分诊**：判断事件类型和严重程度
2. **排查计划**：生成需要收集的证据清单
3. **证据收集**：并行拉取日志、指标、发布记录等
4. **根因分析**：给出可能的根因及修复建议

**如果涉及高风险操作**（如重启服务、回滚版本），您会收到审批通知，需要点击"批准"后才能继续执行。

### 步骤 3：查看结果

处置完成后，在 RCA 报告页面查看：

- 事件时间线
- 根因分析结论
- 处置动作及结果
- 后续预防建议

## 3. 完整功能指南

### 3.1 工单入口（Intake）

**功能说明**：接收故障工单或告警事件，作为整个处置流程的输入。

**操作方法**：

1. 进入工单处理页
2. 选择输入方式：
   - **手动录入**：直接填写故障描述
   - **粘贴告警**：将告警通知的内容粘贴进去
   - **工单 ID**：如果已有关联工单，可输入工单号自动拉取
3. （可选）填写结构化信息：
   - 服务名：明确是哪个服务出问题
   - 环境：生产/预发/测试
   - 严重级别：P1/P2/P3（系统也会自动判断）

**系统行为**：

- 自动抽取关键实体（服务名、错误码、时间等）
- 生成标准化的 `IncidentTicket` 供后续流程使用

### 3.2 智能分诊（Triage）

**功能说明**：对故障进行初步判断，确定排查方向。

**系统自动完成**：

- **判断事件类型**：错误率问题？性能问题？资源问题？依赖问题？发布回归？
- **判断严重级别**：P1（紧急）/P2（重要）/P3（一般）
- **给出初始建议**：推荐从哪个服务、哪个时间窗开始查

**您会看到**：

- 事件类型标签
- 严重程度标识
- 初始排查范围

### 3.3 排查计划（Planner）

**功能说明**：生成结构化的排查步骤，明确需要收集哪些证据。

**系统自动完成**：

- 基于工单内容 + 分诊结果 + 历史知识生成计划
- 列出需要执行的证据收集步骤
- 标注每一步的优先级
- 设定停止条件（何时可以进入诊断）

**示例输出**：

```
1. [高] 拉取 payment-service 最近 30 分钟错误日志，找出 top 错误模式
2. [高] 查询支付接口 p95 延迟指标
3. [中] 检查最近 2 小时内是否有部署变更
4. [中] 检索 runbook 中"支付服务异常"相关章节
5. [低] 查看上下游依赖服务状态
```

### 3.4 证据收集（Investigator）

**功能说明**：并行调用各种工具，收集诊断所需的证据。

**系统自动拉取**：

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 日志 | K8s / ELK | 错误日志、异常堆栈 |
| 指标 | Prometheus / SLB | 5xx 率、延迟、CPU、内存 |
| 发布记录 | 部署平台 | 最近 2 小时的变更 |
| 知识库 | Runbook / FAQ | 历史处置经验 |

**您会看到**：

- 证据面板显示各类证据的摘要
- 每条证据标注来源、时间和可信度

### 3.5 根因分析（Diagnose）

**功能说明**：基于收集到的证据，分析可能的根因。

**系统自动完成**：

- 输出 Top-K 根因候选（带置信度）
- 为每个候选附上支持证据和反证
- 生成修复动作计划

**示例输出**：

```
根因候选 1：数据库连接池耗尽（置信度 85%）
  支持证据：
    - 错误日志出现 "connection pool exhausted"
    - CPU 使用率正常，但活跃连接数达上限
  反证：无
  
修复建议：
  - 步骤 1：扩容数据库连接池（预计影响：小）
  - 步骤 2：检查是否有慢查询（预计影响：小）
  - 回滚预案：恢复原有连接池配置
```

### 3.6 修复执行（Executor）

**功能说明**：执行经过审批的修复动作。

**系统自动完成**：

- 执行低风险自动动作（如清理缓存、调整参数）
- 对高风险动作等待人工审批
- 记录执行结果和耗时

**动作类型**：

| 风险等级 | 示例动作 | 处理方式 |
|----------|----------|----------|
| 低 | 查看日志、查询指标、发送通知 | 自动执行 |
| 中 | 修改配置参数、调整阈值 | 审批后执行 |
| 高 | 重启服务、回滚版本、删除数据 | 强制审批 |

## 4. 审批操作指南

### 4.1 审批触发场景

当 OpsPilot 检测到高风险操作时，会暂停执行并通知您审批：

- **重启服务**
- **回滚版本**
- **修改配置**
- **批量降级**
- **清理队列/缓存**

### 4.2 审批操作

在审批面板，您有以下选择：

| 操作 | 含义 |
|------|------|
| **批准** | 同意执行该动作，继续流程 |
| **拒绝** | 不同意执行，流程终止或转人工 |
| **修改参数** | 同意执行但调整参数（如扩大回滚范围） |
| **要求补证据** | 认为证据不足，让 Agent 补充更多信息 |

### 4.3 审批信息参考

审批时，您可以看到：

- **风险等级**：该动作的预期风险
- **动作参数**：具体要执行什么操作
- **预期效果**：修复后可能达到的效果
- **回滚预案**：如果出问题如何恢复
- **支持证据**：Agent 认为该动作有效的依据

### 4.4 审批超时处理

- 如果审批超时（默认 30 分钟），流程会进入"挂起"状态
- 您可以随时回到工单继续审批

## 5. 查看与分析报告

### 5.1 RCA 报告（Postmortem）

处置完成后自动生成，包含：

1. **事件概述**：什么服务、什么时间、什么问题
2. **时间线**：检测 → 诊断 → 审批 → 执行 → 恢复
3. **根因分析**：最终确认的根因及证据
4. **处置动作**：执行了哪些动作，结果如何
5. **影响范围**：影响的用户、功能、时间
6. **后续预防项**：需要做的改进措施
7. **Agent 表现总结**：哪些地方做得好、哪些不足

### 5.2 Trace 回放

- 可以回看整个处置过程
- 查看每一步调用了哪些工具、返回了什么结果
- 用于复盘和审计

## 6. 常见问题与处理

### 6.1 工单提交后没有反应

**可能原因**：

- 服务名填写不明确，导致无法拉取证据
- 故障描述过于简略

**解决方法**：

- 补充更详细的问题描述
- 明确填写服务名和环境

### 6.2 证据收集时间过长

**可能原因**：

- 查询的时间范围过大
- 目标服务日志量特别大

**系统处理**：

- 会自动降级（缩小时间窗、切换数据源）
- 如果仍然失败会提示转人工

### 6.3 根因分析结果不准确

**可能原因**：

- 证据不足
- 属于新型故障，历史知识无法覆盖

**系统处理**：

- 会给出多个候选（Top-K）而非单一答案
- 每个候选都有置信度供参考
- 您可以结合人工判断选择最终方案

### 6.4 审批被拒绝后怎么办

- 流程会终止或回到人工处理
- 您可以查看拒绝原因，调整后重新提交
- 如果需要紧急处理，可以走人工通道
