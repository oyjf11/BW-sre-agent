# px-doc 浏览器兼容性分析报告

> 版本：2026-04-17 · 分支：`jitword-editor2.2`
> 维度：**Chromium 内核版本**（适用于 Chrome / Edge / Opera / Brave / 360 极速 / QQ 浏览器 / 搜狗 / 微信 WebView / Electron / CEF 等所有基于 Chromium 的运行时）

---

## 0. 为什么要"分情况"讨论

本项目是一个 **Nx monorepo**，同时承载了多个差异极大的功能模块：

- **富文本文档编辑 + 实时协同**（TipTap + Yjs）——能力下限较低
- **表格 / 子文档渲染**（Univerjs 0.15）——**能力下限最高，是整体瓶颈**
- **文档导入导出**（docx / pdf / ofd / html）——中等
- **图表 / 思维导图 / 流程图**（ECharts / mind-elixir / vue-flow）——中等
- **AI 写作辅助、页面预览、历史版本对比**——轻量

其中只有 **Univerjs** 官方声明了硬性的 **Chromium ≥ 100** 要求。其余模块的实际下限远低于 100。因此"项目到底支持到哪个内核"**完全取决于你启用了哪些模块**，必须分情况说明。

## 2. 分情况支持矩阵

### 情况 A：**完整功能**（含 Univer 表格/子文档） 🔴 当前默认形态

> 适用场景：正式生产部署，用户会在文档中插入/编辑 Univer 表格、子文档等高级对象。

| 项 | 要求 |
|---|---|
| **最低 Chromium 内核** | **100**（2022-03） |
| **推荐 Chromium 内核** | **109+**（2023-01，Windows 7/8 最后支持版，企业装机常见） |
| **舒适 Chromium 内核** | **118+**（2023-10） |
| 等价 Chrome 版本 | 100+ |
| 等价 Edge 版本 | 100+ |
| 等价 Firefox | 100+ |
| 等价 Safari | 15.4+ |
| 瓶颈原因 | [apps/editor/package.json:55-66](../apps/editor/package.json#L55-L66) 的 `@univerjs/*@0.15.2` 官方最低线 |

**说明**：即使其它依赖可以跑到 Chromium 87，但只要用户点开表格/Univer 子文档，页面就会报错或渲染异常。所以只要启用 Univerjs，对外口径就必须是 **≥ 100**。

---

### 情况 B：**禁用 Univerjs 后**（纯 TipTap 文档 + 协同 + 导入导出） 🟡

> 适用场景：面向低版本客户定制部署；通过按路由懒加载 + 运行时能力探测，把 Univer 入口隐藏；或用 Luckysheet/自研轻量表格替代。

| 项 | 要求 |
|---|---|
| **最低 Chromium 内核** | **92**（2021-07） |
| **推荐 Chromium 内核** | **98+**（支持 `structuredClone`） |
| 瓶颈原因 | pdfjs-dist 4.x（OFD/PDF 预览）→ Chromium 92；若同时关闭 PDF 预览，可继续下探到 **87** |
| 必做改动 | ① 移除或路由级懒加载 [apps/editor/package.json:55-66](../apps/editor/package.json#L55-L66) 的 `@univerjs/*`<br>② 在 [apps/editor/vite.config.js](../apps/editor/vite.config.js) 的 `optimizeDeps.include` 中移除 Univer<br>③ 运行时若检测到 Chromium < 100 则屏蔽表格/子文档入口 |
| 估工 | ~20 小时 |

---

### 情况 C：**禁用 Univer + PDF/OFD 预览**（纯编辑 + 协同） 🟢

> 适用场景：仅作为在线编辑器使用，不内嵌 PDF 预览/OFD 阅读器。

| 项 | 要求 |
|---|---|
| **最低 Chromium 内核** | **87**（2020-11，Vite 5 默认线） |
| **推荐 Chromium 内核** | **90+** |
| 瓶颈原因 | Vite 5 默认 `'modules'` 目标；shiki WASM 需 85；`@vue-flow` 需 87 |
| 必做改动 | 情况 B 的改动 + 禁用 PDF/OFD 预览模块（或仅服务端导出、前端不渲染） |
| 估工 | ~30 小时 |

---

### 情况 D：**只读预览模式**（不编辑、不协同，仅查看已渲染结果） 🟢

> 适用场景：分享链接、只读预览页、政务归档查看器；核心是单向渲染 HTML + 静态资源。

| 项 | 要求 |
|---|---|
| **最低 Chromium 内核** | **80**（2020-02） |
| 瓶颈原因 | `?.` / `??` 在产物中直接输出（TS target=ES2020 的包）；ECharts 5.5 |
| 必做改动 | ① 单独产出一份"预览端"bundle<br>② Vite 显式 `build.target: ['chrome80','edge80','firefox78','safari13.1']`<br>③ 仅保留查看/导出下载能力 |
| 估工 | ~40 小时 |

---

### 情况 E：**最激进下探**（加 legacy 插件 + polyfill） 🟠 不推荐

> 适用场景：必须覆盖 Windows 7 + 旧 360 安全浏览器兼容模式等历史系统。

| 项 | 要求 |
|---|---|
| **最低 Chromium 内核** | **70**（2018-10） |
| 必做改动 | ① 所有前面情况改动<br>② 引入 `@vitejs/plugin-legacy` + `core-js@3`<br>③ 补 `globalThis` / `structuredClone` / `queueMicrotask` 等 polyfill<br>④ 补 `Promise.allSettled` 等 API polyfill<br>⑤ Bundle 体积 +150~200 KB |
| 估工 | 100+ 小时 |
| 风险 | Vue 3.5 / TipTap 2 / Yjs 的某些运行时路径仍可能在 Chromium 70–76 上出边缘 bug，需逐项验证 |

---

### 情况 F：**IE 11 / EdgeHTML**（非 Chromium） ❌

| 项 | 结论 |
|---|---|
| 支持 | **完全不支持**、**不可能支持** |
| 原因 | Vue 3、TipTap、Univer、Yjs、ProseMirror 全部不兼容 IE；项目零 polyfill；WebSocket/Proxy/ResizeObserver/WASM 等核心依赖 IE 全不支持 |
| 建议 | 遇到 IE UA 直接跳转到"请使用 Chromium 100+ 内核浏览器"的引导页 |

## 5. 落地建议

### 5.1 显式声明浏览器目标

在 [apps/editor/package.json](../apps/editor/package.json) 添加：

```json
"browserslist": [
  "Chrome >= 100",
  "Edge >= 100",
  "Firefox >= 100",
  "Safari >= 15.4"
]
```

在 [apps/editor/vite.config.js:145](../apps/editor/vite.config.js#L145) 显式：

```js
build: {
  target: ['chrome100', 'edge100', 'firefox100', 'safari15.4'],
  // ...
}
```

### 5.2 入口处内核探测

项目已引入 [@fingerprintjs/fingerprintjs](../apps/editor/package.json#L15)，可复用其 UA 解析逻辑。在登录页 / 入口页执行：

```js
const m = navigator.userAgent.match(/Chrom[ei]um?\/(\d+)/i)
const chromiumVersion = m ? parseInt(m[1], 10) : 0
if (chromiumVersion && chromiumVersion < 100) {
  // 跳转到升级引导页 / 切极速模式引导
}
```

配合 **情况 B / C** 的部署版本，探测到 < 100 时可保留访问、但**屏蔽 Univer 入口**并给出提示条。

### 5.3 对外宣传口径（建议）

> px-doc 推荐在 **Chromium 109+ 内核**的浏览器中使用（Chrome/Edge/Opera 109+、360 极速浏览器 22+、QQ 浏览器 11+、搜狗高速 12+ 等）。最低支持 **Chromium 100 内核**；Safari 用户需 **15.4+**。不支持 IE 全系列及旧 Edge。

### 5.4 差异化部署策略

如确有低版本客户需求，可基于本文档提供两套产物：

- **Full 版本**（默认）：对应情况 A，≥ Chromium 100
- **Lite 版本**：对应情况 B/C，≥ Chromium 87/92，通过构建期条件剥离 Univer / pdf 预览

在 [apps/editor/vite.config.js](../apps/editor/vite.config.js) 通过环境变量（如 `VITE_ENABLE_UNIVER`）控制是否把 `@univerjs/*` 打入 bundle。

---

**报告结论**：
- **默认形态（启用 Univer）**：最低 **Chromium 100**，推荐 **109+**
- **不启用 Univer**：最低可降至 **Chromium 92**（关闭 PDF 预览后可至 87）
- **只读预览**：最低 **Chromium 80**
- **IE 不支持，不建议尝试下探到 Chromium 70**
