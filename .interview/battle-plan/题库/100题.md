# AI 应用开发面试题：Agent 在大型企业落地的 100 道题

## 一、需求与场景定义（1-10）

1. 如何判断一个企业业务场景适合用 Agent 而非传统规则引擎或 RPA？
2. 当业务方提出"做个智能客服 Agent"时，如何拆解模糊需求并定义可交付的边界？
3. Agent 项目的 ROI 如何量化？哪些隐性成本容易被忽视？
4. 如何区分"必须用 LLM Agent"和"用普通 workflow+少量 LLM 调用即可"的场景？
5. 面对"全自动"和"人机协同"两种诉求，如何为企业选择落地形态？
6. 如何在 POC 阶段设计能说服决策层的成功标准（成功率、覆盖率、降本数据）？
7. 多个部门提出相似 Agent 需求时，如何设计可复用的平台而非烟囱式重复建设？
8. 如何评估一个任务的"长尾复杂度"，避免上线后大量 case 无法覆盖？
9. 业务流程频繁变更的场景下，Agent 设计如何保持可维护性？
10. 如何向非技术高管解释 Agent 的能力边界，管理过高预期？

## 二、架构设计（11-25）

11. 单 Agent、多 Agent、分层 Agent 架构各自适用什么企业场景？
12. 多 Agent 系统中如何设计编排层（orchestrator）与子 Agent 的职责划分？
13. 如何在 ReAct、Plan-and-Execute、Reflexion 等范式间做选型权衡？
14. 状态管理：长流程 Agent 的会话状态、任务状态应该存在哪里，如何保证一致性？
15. 如何设计 Agent 的记忆系统（短期/长期/情景/语义记忆）在企业级数据量下的存储与检索？
16. Agent 与现有企业系统（ERP、CRM、工单系统）集成的常见架构陷阱有哪些？
17. 如何设计工具调用层，使新增工具不需要改动核心代码？
18. 同步调用与异步任务队列在 Agent 架构中如何取舍？
19. 如何设计支持断点续跑（resume）的长时任务 Agent？
20. 多租户场景下如何做数据隔离与资源配额？
21. 如何在架构上支持模型热切换（不同任务用不同模型）？
22. Agent 的"控制流"由 LLM 决定还是由代码决定，如何在可控性和灵活性间平衡？
23. 如何设计降级方案：LLM 不可用或超时时系统如何兜底？
24. 边缘部署 vs 云端部署，对延迟敏感的企业 Agent 如何选择？
25. 如何设计 Agent 间的通信协议（共享内存、消息总线、A2A 等）？

## 三、上下文与 RAG 工程（26-38）

26. 企业知识库动辄百万文档，如何设计分层检索避免上下文爆炸？
27. 检索召回率低时，如何系统化定位是切分、嵌入、还是 query 改写的问题？
28. 如何处理结构化数据（数据库、表格）与非结构化文档混合的检索？
29. 文档频繁更新的场景，如何设计增量索引与缓存失效策略？
30. 上下文窗口有限时，如何对历史对话和检索结果做压缩与优先级排序？
31. 如何解决多文档冲突信息（不同版本制度文件）的检索消歧？
32. 权限敏感场景下，如何保证检索结果遵守用户的数据访问权限（行级/文档级权限）？
33. 如何评估 RAG 质量？哪些指标（召回、精度、忠实度、相关性）该如何度量？
34. Agent 多轮调用中如何避免上下文随轮次累积导致成本和延迟激增？
35. 长文档问答中"大海捞针"问题如何缓解？
36. 如何设计混合检索（向量+关键词+元数据过滤）并调权？
37. GraphRAG 相比传统 RAG 在企业知识场景的优劣？
38. 如何处理多模态内容（图表、扫描件、PDF 表格）的检索与理解？

## 四、工具/Function Calling（39-48）

39. 工具数量增多（几十上百个）时，如何让模型准确选对工具？
40. 如何设计工具的 schema 描述以降低模型误调用率？
41. 工具调用参数幻觉（编造不存在的 ID/字段）如何防范？
42. 如何处理工具调用失败的重试、回退与错误信息回传给 LLM？
43. 危险操作（删除、转账、发邮件）的工具如何设计审批与确认机制？
44. 如何对工具调用做权限校验，防止 Agent 越权操作？
45. 并行工具调用与串行依赖工具调用如何编排？
46. 如何对接老旧系统中没有 API、只有界面的操作（UI 自动化的取舍）？
47. 工具返回数据量过大时如何处理（分页、摘要、截断）？
48. 如何设计工具的幂等性以应对 Agent 重复调用？

## 五、可靠性与质量（49-60）

49. 如何系统化降低 Agent 的幻觉？有哪些工程手段？
50. 多步任务中错误会累积放大，如何设计中间校验点（checkpoint/validation）？
51. 如何让 Agent 知道"自己不知道"并主动转人工或澄清？
52. 非确定性输出如何做回归测试，保证迭代不退化？
53. 如何构建 Agent 的评估集（golden set）并持续维护？
54. LLM-as-judge 做自动评估的可靠性问题与校准方法？
55. 如何检测和处理 Agent 陷入死循环或无限重试？
56. 输出格式不稳定（JSON 解析失败）如何工程化保证结构化输出？
57. 如何设置任务的最大步数/预算上限并优雅终止？
58. A/B 测试一个 Agent 改动时，如何控制变量与统计显著性？
59. 如何区分"模型能力不足"与"prompt/工程问题"导致的失败？
60. 生产环境中如何采样和复盘失败 case 形成改进闭环？

## 六、评估与监控（61-70）

61. Agent 上线后应该监控哪些核心指标（成功率、延迟、成本、转人工率）？
62. 如何做端到端的可观测性（trace 每一步的输入输出、工具调用、token）？
63. 如何在没有明确 ground truth 的开放任务上衡量质量？
64. 用户满意度与任务成功率不一致时如何归因？
65. 如何搭建线上 badcase 的自动发现与告警机制？
66. 如何评估多 Agent 系统中是哪个环节出了问题（责任归因）？
67. 离线评估表现好但线上表现差，常见原因有哪些？
68. 如何度量 Agent 的"自主性程度"并据此调整人工介入策略？
69. 评估成本本身很高时（需人工标注），如何降本？
70. 如何建立模型/prompt 版本与线上效果的关联追踪？

## 七、安全与合规（71-82）

71. 如何防范 Prompt Injection，尤其是间接注入（来自检索文档或网页）？
72. Agent 能调用工具，如何防止被诱导执行恶意操作（如数据外泄）？
73. 如何做输出的内容安全审核（敏感信息、违规内容）？
74. PII 数据在 Agent 链路中如何脱敏、最小化暴露给第三方模型？
75. 数据出境/数据驻留合规要求下，模型与数据如何部署？
76. 如何满足审计要求：Agent 每个决策可追溯、可解释、可复现？
77. 越权访问防护：如何保证 Agent 以"调用者身份"而非"超级权限"操作？
78. 如何防止训练/RAG 数据中的机密信息通过模型泄露？
79. 行业监管（金融、医疗）对自动化决策的限制如何影响 Agent 设计？
80. 如何处理模型输出的法律责任归属问题？
81. 多 Agent 系统中信任边界如何划分，外部 Agent 如何隔离？
82. 如何应对对抗性用户（越狱、滥用）的防护策略？

## 八、性能、成本与扩展（83-92）

83. Agent 单次任务 token 成本高，有哪些系统化降本手段？
84. 多步 Agent 的端到端延迟如何优化（并行化、缓存、模型分级）？
85. 如何用小模型/大模型路由（router）平衡成本与质量？
86. 提示词缓存（prompt caching）在 Agent 场景如何有效利用？
87. 高并发下如何应对 LLM API 的限流（rate limit）与排队？
88. 如何做容量规划，预估 Agent 系统的 QPS 与成本？
89. 推理结果缓存的命中率如何提升，又如何避免缓存脏数据？
90. 自建推理 vs 调用 API 的成本临界点如何测算？
91. 如何在保证质量的前提下做模型蒸馏或微调以降本？
92. 流式输出在 Agent 多步场景下如何实现以改善体验？

## 九、组织、工程与落地（93-100）

93. Agent 项目从 POC 到生产，最常见的"死亡之谷"是什么，如何跨越？
94. 如何建立 Prompt 和 Agent 逻辑的版本管理与 CI/CD 流程？
95. 数据飞轮：如何利用线上交互数据持续改进 Agent？
96. 团队需要哪些角色（提示工程、ML、后端、领域专家）如何协作？
97. 如何说服企业内部对"不确定性系统"建立信任并推动采用？
98. 用户不信任或不会用 Agent，如何做交互设计与变更管理？
99. 模型供应商锁定风险如何规避，如何做多模型抽象层？
100.  如何制定 Agent 的迭代节奏，平衡快速上线与稳定可靠？
