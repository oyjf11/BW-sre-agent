# 设计：题库全量答案重写

## 任务信息

- 任务名称：题库-全量答案-实战版
- 创建日期：2026-06-15
- 输出文件：`题库-全量答案-实战版.md`
- 输入：`题库-整合版.md`（676 题） + `母题骨架-一页纸.md`（32 母题） + battle-plan 全员实战文章

## 目标

将题库 676 题全部重写为「一句核心 → 展开 2-3 层 → 项目印证」格式，每条答案锚定一个 battle-plan 实战文章或三个项目（OpsPilot / 前端测试域 / 应用生成平台）的具体情节。

## 非目标

- 不改动原《答案.md》《高危题标准答案.md》《母题骨架-一页纸.md》
- 不新增题目
- 不修改题目原文

## 三轮流水线

### Round 1：母题库（先深后广）

32 个母题各写完整答案（3-4 层展开 + 项目锚点 + 素材路径）。

输出位置：文件前半部分，以「# 母题库」为一级标题，每道母题二级标题。

每道母题答案结构：

```markdown
## M{N}：{母题名}

**Kernel**：{核心一句话，30 字以内}

**展开**：
### Layer 1 — 为什么 / 背景
...
### Layer 2 — 怎么做 / 机制
...
### Layer 3 — 坑 / 边界 / 追问预判
...
### Layer 4 — 同类变式（可选，高频母题加）

**项目印证**：
- OpsPilot：{具体情节}
- 前端测试域：{具体情节}（如适用）
- 应用生成平台：{具体情节}（如适用）

> 📎 素材：{battle-plan 文章路径}
```

### Round 2：逐题映射（广覆盖）

676 题逐题产出轻量答案。每题流程：

1. **题眼归因**：判断对应哪个母题（允许一对多，比如一道题同时涉及 M1 + M7）
2. **生成答案**：格式如下

```markdown
### Q{N} {题目原文}

**一句核心**：{kernel，30 字以内}

**展开**：
- {第一层}
- {第二层}
- {第三层（如有坑/追问）}

**项目印证**：{具体锚点}

> 🏷️ 母题：M{N}
```

核心逻辑：展开部分的实质内容来自母题 kernel，按题目具体问法选择性展开。项目印证优先引用最相关的那个项目。

### Round 3：收口

1. **🅰️ 组精修**：A1–A10 + M21–M23 + RAG+Agent 结合题（9 题）逐题深修，确保开口 5 秒能出
2. **同类题统一**：同一母题下多道题目的表述保持一致，kernel 不打架
3. **去重**：语义完全重合的题目标注「→ 同 Q{N}」

## 格式规范

每条答案必须含以下三项，缺一不可：

1. **一句核心**：加粗，30 字以内，面试开口第一句
2. **展开**：2-3 条要点，每条一行，bullet 列表
3. **项目印证**：必须以 OpsPilot / 前端测试域 / 应用生成平台之一的具体情节结尾，不能泛泛而谈

## 素材引用规则

每条答案必须注明所引用的 battle-plan 文章（如有），格式：
```
> 📎 素材：{相对路径}
```

无对应素材的标注 `> 📎 素材：待补充`

## 验收标准

- [ ] 676 题每题都有答案
- [ ] 🅰️ 组每题可 90 秒内口述完毕
- [ ] 每条答案含「一句核心 + 展开 + 项目印证」三项
- [ ] 项目印证不空洞——必须是具体情节，不能是「在我们的 Agent 项目中我们也做了这个」
- [ ] 同类题表述统一，无矛盾
- [ ] 无「TODO」「待补充」占位符（除素材引用外）

## 风险与回滚

- **风险**：676 题全量重写工作量大，单次会话可能无法完成。缓解：按模块分批，每完成一个模块落盘一次
- **回滚**：输出为新文件，不影响原有《答案.md》《高危题标准答案.md》
