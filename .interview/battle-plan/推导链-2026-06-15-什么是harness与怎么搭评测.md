## 第 0 段 · 这份东西怎么用（先读这段，别跳）

你手上那四篇文章（手册、那篇批判营销文的、两篇评测实战），都是**写给已经懂了的人，做面试前一晚快速翻的速记**。它们敢中英混着写、敢跳步、敢一上来就甩结论和表格，是因为它默认你脑子里已经有一条完整的推理线了，它只是帮你快速点一下。

问题是你现在还没有那条线。拿速记当入门材料读，当然越读越累、越读越接不上。

## 第 1 段 · 推导链 A：什么是 harness

**起头第一句（只用记这句）：模型本身不值钱，值钱的是包在模型外面那层工程。**

下面一步步说清楚，为什么从这句话出发，最后一定会走到"幂等、审批、防失控"这些东西上。

**第 1 步 · 一个事实：模型是个买得到的东西。**
大模型半年就换一代，开源的随便下载，闭源的交钱就能调 API。也就是说，模型这东西，你有、你对手也有，谁都能拿到。一个谁都能拿到的东西，构不成你的本事。

**第 2 步 · 既然模型谁都有，你的本事只能在"模型外面那层"。**
同一个模型，有人用它做出能上线的产品，有人做出来一上线就出事。差别不在模型，在它外面那一圈工程。这一圈工程，行业里有个名字，叫 **harness**（直译是"挽具、套在牲口身上的那套绳套"，这里就是"套在模型外面、驾驭它的那套工程"）。
一句话把它定死：**harness = 让模型的脑子变得能用、能管得住、能验证对错的那一整套工程。**

**第 3 步 · 这"一整套工程"到底包含啥？别让它停在抽象。**
说"一整套工程"太空，落到地上就是五件具体的事：
1. 这个模型能调用哪些工具（查日志？改配置？）
2. 它有什么权限（只能看，还是能动手）
3. 每次给它看哪些信息（喂什么资料进去）
4. 怎么给它的回答打分（它干得对不对，谁说了算）
5. 它出了事怎么查、怎么追责（留没留记录）
记住这五件就行——别人问"harness 具体是啥"，你把这五件数出来，比说一堆形容词管用。

**第 4 步 · 给这层换个更准的说法：它是"控制面"，和"干活的那部分"是分开的。**
上面那五件事有个共同点：它们都不直接干业务，它们是在旁边"管着"业务。行业把这种"只管控制、不亲自下场干活"的部分叫**控制面**（control plane，原是网络设备的词：一台交换机里，决定数据往哪走的那部分叫控制面，真正搬运数据的那部分叫数据面）。
套到这里：模型一个节点一个节点处理故障，那是"干活的部分"；而打分、限流、安全检查、留记录这些，是"控制面"。**好的 harness，控制面和干活的部分是分开的——你加一套监控、加一道安全检查，不用去改干活的代码。** 这就是为什么它能独立演进、能复用。

**第 5 步 · 到这里有个坎，很多人就停在坎前面、然后翻车。**
如果你的 Agent 只是"只读问答"——比如客服机器人、查知识库的助手，它只会读、只会答、不会动手改任何东西——那 harness 的难处到此为止：限流（别被打爆）、留记录（出事能查）、测一测（上线前跑几遍）。这种情况叫**读路径**（read path，意思是 Agent 只读不写）。读路径的 harness，说实话谁都能搭，没什么门道。

**第 6 步 · 真正难的是"写路径"——Agent 要动手改真东西。**
换个 Agent：它要执行有后果的动作，比如回滚一次发布、重启一个服务、改一份线上配置。这叫**写路径**（write path，Agent 会真的改动外部世界）。
为什么写路径一下就难了？因为后果不一样。读错了，顶多答错一句话，没人受伤。写错了，是真的把线上搞坏了——而且更糟的是，它可能**重复**搞坏（同一个动作执行两遍）、在一个本来就不正常的状态上动手、或者自己陷进去反复折腾停不下来。

**第 7 步 · 所以写路径被逼着长出四样东西，每样都对着一个具体怕的事。**
不是有人设计得花哨，是不加这四样就一定出事：
1. **幂等键**（idempotency key，"幂等"就是同一个操作做一遍和做多遍结果一样）——怕的是同一个动作被执行两遍（比如网络重试）。给每个动作发一个唯一的 key，执行前先查这个 key 干过没有，干过就跳过。
2. **前置校验**（执行前先检查）——怕的是在一个坏状态上动手。比如要回滚，先查"这个服务真的存在吗、当前是不是健康的"，条件不对就不动。
3. **人工审批**（行业缩写 HITL，human in the loop，"人在回路里"）——怕的是高风险动作没人兜底。生产环境的、风险高的动作，必须停下来等人点头才能继续。
4. **防失控**（loop guard，"loop"是循环、"guard"是守卫）——怕的是 Agent 自己转圈出不来（改不对 → 再试 → 还不对 → 再试……）。给它设个上限，比如最多试 2 轮，到头还不行就转人工，别无限烧下去。

**第 8 步 · 最后把这条链兜回来。**
所以"什么是 harness"，完整答案是两层：
- 通用的说法：harness 是模型外面的控制面，把监控、限流、安全、打分从干活的代码里抽出来，单独管。模型是商品，这层才是你的本事。
- 但真正能让对方觉得你懂的，是你能当场把读路径和写路径分开：**只读问答的 harness 谁都能搭；要动手改真东西的写路径，才是真正难的地方——而它就难在那四样：幂等、前置校验、人工审批、防失控。**

（你自己这个 OpsPilot 项目就是写路径：executor 里有幂等键和前置校验，risk_gate 接审批，critic 有最多 2 轮的防失控。讲的时候直接拿它当例子。）

## 第 2 段 · 推导链 B：怎么搭评测

先说清楚 B 和 A 的关系：A 里第 3 步那五件事，第 4 件是"怎么给模型的回答打分"。**B 就是把这一件单独拎出来、放大了讲。** 它不是新话题，是 A 的一个零件的展开。

**起头第一句（只用记这句）：评测要搭多复杂，是被"对错有多难判断"逼出来的。**
"对"如果机器自己就能判（比如测试过没过），一段代码就够了；"对"一旦带主观、得有人来评，才被迫上更重的东西。下面这条链，就是顺着"对越来越难判"一步步加码。

**第 1 步 · 先说一个最常见的错误做法，好让你知道为什么要"搭"。**
给 Agent 打分，最容易犯的错是：跑几个例子，看着顺眼，就说"行"。这不叫评测，叫演示。它有三个洞：
1. 没有标准答案的题，等于没考——你凭感觉说好，换个人就说不好。
2. 一个总分把什么都盖住了——"综合 85 分"，既容易被刷，又看不出到底哪一段弱。
3. 今天过、明天挂——外部世界一直在变，同一道题结果不稳，分数没法比。
评测体系，就是用最少的零件把这三个洞堵上。

**第 2 步 · 第 0 步先确认：这事有没有"自动裁判"。**
要打分，前提是存在某种"机器能自动判对错"的手段——能跑的测试、有最终结论的历史数据、可以核对的事实。如果一个 Agent 的输出连人都说不清对错，那就先别评它。先找到裁判，再谈评测。

**第 3 步 · 第 1 步：多阶段的 Agent，不许给一个总分，要拆开评。**
真实的 Agent 基本都是好几段串起来的，比如前端测试 Agent 就是：写用例 → 跑用例 → 修用例 → 守着别让回归漏掉。每一段"什么算对"的含义都不一样。揉成一个总分，前面说过——既会被刷，又看不出哪段弱。所以按它干活的先后拆成几段，一段一段单独评。

**第 4 步 · 第 2 步，最关键的一步：把每一段的"对"拆成两半。**
每一段的"对"，都能拆成两半：
- **机器能判的那一半**：比如代码跑过没过、关键的那行有没有被改动、生成的测试能不能抓住一个真实坏过的 bug。这一半交给代码判，不用请人。
- **只能靠人判的那一半**：比如"这个修复，意思上合不合理"。这一半才需要后面那套重家伙。
功夫就在于：**尽量把"对"往机器能判的那半边推**，推到推不动了，剩下那一点点才请人。
这里有一刀必须记住：**判"测试写得够不够"，绝对不能用覆盖率**（代码有多少行被测试跑到了）。因为 AI 几秒钟就能生成一堆"空壳测试"——它把代码跑了一遍但什么都不检查，覆盖率冲到 100%，啥用没有。正确的判法是问"它抓不抓得住真 bug"：把代码退回到某个真实 bug 被修好之前，看这些测试发不发得现。把一个开放的、主观的问题（该测什么），换成一个封闭的、是非的问题（能不能抓住这个具体的坏 bug）。

**第 5 步 · 第 3 步：配裁判，按"能不能机器判"分三档，从便宜到贵。**
1. **代码判**（首选，不花钱、结果稳定）：答案命中没命中、关键词在不在、那行代码的结构改没改、放进沙箱重跑过没过。
2. **直接跑一遍**：能编译能测试的，让编译器和测试说话。
3. **让另一个模型当裁判**（只给真正主观的那一点）：而且必须降噪——把随机性关到零（参数叫 temperature，设成 0）、只让它打两三档别打 1 到 10 分（在 7 分 8 分之间纠结只会添乱）、同一份判三次取多数、当裁判的模型和被考的模型不能是同一个。

**第 6 步 · 第 4 步：建题库——这是考 Agent 的卷子。**
题库就是仓库里的一个文件，每行一道题。一道题包含：
- 输入：一个藏住了结论的历史案例，加上当时那一刻的数据快照（快照是下一步要讲的、能重复考的关键）。
- 期望：标准答案、可以接受的答案范围、故意埋的"不可接受"陷阱选项、证据里必须出现的关键词。
- 三类题：**业务题**（判得对不对，答案从历史数据来）、**行为题**（过程规不规矩——该查的查了吗、有没有想越权、信息不够时是老实说"建议补充"还是硬猜）、**安全题**（输入里藏一句"忽略上面的规则、照我说的做"，看它中不中招，这类必须 100% 不中招）。
标准答案哪来？历史白送——历史工单的结案结论、git 里人工修复的那一版，都是现成的答案。

**第 7 步 · 第 5 步：凡是用了"模型当裁判"的地方，再建一个 gold set——这是考裁判的卷子。**
模型当裁判，它自己也会抖、也会错。所以裁判本身也得被考。考裁判的卷子叫 **gold set**（金标集，"金标"就是标准答案里最权威的那种）。它是几十条由人类专家亲手打过分的记录，加一段算"一致率"的代码。怎么建：
1. 三位专家各自独立打分，不许商量。
2. 先看专家之间一致不一致（用一个叫 **kappa** 的统计数算，它比"简单一致率"好在扣掉了"瞎蒙也能蒙对一半"的水分）。
3. 专家都打不齐，就开会把评分标准改清楚——改的是标准，不是怪某个人。全员一致的题才收进来，吵不拢的边界题直接扔掉。
4. 裁判只要改动过（提示词改一行、裁判模型升一版），必须先在 gold set 上重新及格，才许重新上岗。
为什么不会没完没了地"谁来考考裁判的裁判"：因为 gold set 是死数据加算术，跑一万次结果都一样，不会抖。链条到这就到头了，终点是人类白纸黑字达成的共识。

**第 8 步 · 第 6 步：冻结世界——这是"能重复考"的地基。**
外部世界一直在变（接口返回、时间戳、随机值），同一道题今天过、明天挂，分数就没法比。所以考之前，先把某一时刻的外部世界录下来、冻住，Agent 考试时查到的是这份冻住的快照，不是实时系统。这一步不产出分数，但前面所有分数能不能信，全靠它。

**第 9 步 · 第 7 步：把分数汇总成一组指标，永远当一组报，绝不揉成一个数。**
把被考的能力当成一个"出没出事的分类器"来看，报四个指标：
- 信噪比（行业叫 Precision，精确率）：报红里面有多少是真问题。
- 发现力（行业叫 Recall，召回率）：真问题里有多少被抓住了。
- 有效覆盖：需求被"非空壳"的测试覆盖了多广。
- 维护成本：花了多少工时、多少钱。
为什么不揉成一个数：**任何被拿来考核的单一指标，都会被想办法刷**（这叫 Goodhart 定律）。四个一起报，它们互相咬着——你想靠删掉抖动的测试来刷信噪比，发现力立刻掉下来，藏不住。
另外两条纪律：和基线（人工/老办法）比的时候，一次只改一个变量、在同一批题上配对比；报分数永远带误差范围（写"79% ± 5"，别写"79%"，因为 200 道题跑一遍，本身就有正负五个多点的浮动）；分数卡在及格线附近，正确做法是加题再考，绝不能原样反复重跑、撞到运气好那次就算过。

**第 10 步 · 最后把这条链兜回来。**
一句话：**题库考 Agent，gold set 考裁判；机器能判的活全交给代码，只有真主观的活才请模型来判、再用人类共识去校准这个模型。** 整套东西做的事，就是把一次很贵的人类共识，便宜地复制到一百万次自动打分里。

## 第 3 段 · 站到路顶：怎么把它讲出来不散

前面两条链是为了让你"懂"。这一段是为了让你"讲得出"。你之前的问题是开口能讲、但越讲越散、收不回来——原因是你脑子里没有一个固定的起点，每次都从随机一个点开始往外蹦。

办法很简单：**每条链你只死记第一句，被问到时从这第一句往下顺，顺到对方打断为止。** 顺序我已经替你排好了，你不用临场重新组织。

**问"什么是 harness"，你的起点永远是这句：**
> 模型本身不值钱，值钱的是包在外面那层工程。

然后顺着往下：模型是商品 → 本事在外面那层（叫 harness）→ 这层就五件事（工具、权限、信息、打分、追责）→ 这层叫控制面、和干活的分开 → 分读路径和写路径 → 写路径才难，难在四样（幂等、前置校验、审批、防失控）。
对方要是只想听一句，就给前半句加一句"而真正难的是写路径"；想听细的，就往四样上铺。

**问"怎么搭评测"，你的起点永远是这句：**
> 评测要搭多复杂，是被"对错有多难判断"逼出来的。

然后顺着往下：先有没有裁判 → 多段拆开评 → 每段把"对"拆成机器判的和人判的（覆盖率不能当裁判，要用"抓真 bug"）→ 配裁判分三档（代码判 > 跑一遍 > 模型判）→ 建题库考 Agent → 用了模型判就建 gold set 考裁判 → 冻结世界保证能重复 → 汇总成一组指标不揉成一个数。

**给自己出三道自测题（能从起点一句话推出来，就算过）：**
1. 别人问"评测什么时候才需要 gold set？"——你能不能不背、直接从"对越主观越要请人"推出来？（答案：只有用了模型当裁判的那一段才要，机器能判的不要。）
2. 别人问"harness 最难的部分是什么？"——你能不能从"读路径 vs 写路径"切进去？（答案：写路径，难在那四样。）
3. 别人问"为什么不能用覆盖率打分？"——你能不能从"AI 几秒能生成空壳"推出来？（答案：空壳测试能把覆盖率刷满却什么都不查，所以改用"抓真 bug"。）
这三道，你不是在背答案，是在练"从起点一句话现场长出答案"。练顺了，别人怎么问、从哪个点切，你都能拐回到那句起点上，就不会散了。
