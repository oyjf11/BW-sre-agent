分块解读

① + ④：命中率全部打满 100%

10 条 case 三路全 ✓，A 基线就 100%。说明：

- 测试集对裸向量已经太简单（用户口语和子块场景描述语义距离不够大）
- 改写和 rerank 在"命中率"维度无地施展——基线已经天花板，没有提升空间
- 归因 +0 pp 是正确的，不是 bug  


这是评测设计的局限，不是手段的局限：要看出 rerank  
 的命中率收益，得换更难的测试集（口语更绕、范式相互更接近、加干扰类目）。

③ 这才是本次的真戏 —— margin 翻 2.4 倍

 │ 方案 │ 平均 margin │ 含义 │
├───────────┼─────────────┼──────────────────────────────────────────┤

├───────────┼─────────────┼──────────────────────────────────────────┤

生产意义：margin 大意味着可以设置一个阈值（比如 0.4）做"低置信度拒答 / 转人工"。A 路 margin  
 0.21，没法拒答；C 路 margin 0.51，可以加 gate。这一栏才是 rerank 上生产的真正价值。

⚠️ 一个评测小毛病要点出来

第 ③ 行的 "平均 Top1 分数"：

┌──────┬────────────────────────┐  
 │ 方案 │ 平均 Top1 分数 │  
 ├──────┼────────────────────────┤
│ A │ 0.719 │
├──────┼────────────────────────┤
│ B │ 0.787 │  
 ├──────┼────────────────────────┤
│ C │ 0.684 ← 看着反而降了？ │  
 └──────┴────────────────────────┘

别上当，这是分数尺度不同造成的视觉错觉。 A/B 用的是 cosine 相似度（embedding 内积，0~1），C  
 用的是 gte-rerank 的 relevance_score（cross-encoder  
 输出的相关性分，不是相似度，尺度天然不一样）。直接横比 0.684 vs 0.787  
 没有意义——就像比"苹果重量"和"苹果甜度"。

margin 横比为啥又有意义？ 因为 margin = top1 - top2 是同一尺度内的相对差，C 的 margin 反映  
 rerank 自己尺度下"敢下结论"的程度，跟 A/B 的 margin 物理含义可比。

评测建议：以后跑这种报表，应该在 ③ 这一栏给 C 的"平均 Top1 分数"加个脚注  
 (不同尺度,仅供参考)，或者干脆只保留 margin 那一行。我可以顺手改一下。

个别 case 印证

第 2 条 "卖东西的网站"：A=0.59、B=0.69、C=0.43。看着 C 倒退最多。但实际 C 的 0.43 是 rerank  
 给 P_ecommerce 的绝对相关性，而 P_ecommerce 的 top2 估计只有 0.1 几——margin
反而最大。这就是为什么 ③ 那栏 C 的平均 margin 飙升。

元数据过滤那段也是同款故事

 cosine，这三个分数大概率挤在 0.7~0.85 之间，margin 0.1 左右，根本不敢加阈值做拒答。
