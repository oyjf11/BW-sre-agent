用你的具体场景说，不抽象。

你有 500 条工单，每条长这样：

```
text: "调用支付接口返回 502，持续 3 分钟"
label: "接口报错5xx-服务器不可用"
```

---

## 以 SetFit 为例，完整走一遍

### 第一阶段：对比学习调整 encoder

SetFit 先从你的 500 条里**自动构造训练对**：

```
正对（同类）：
  ("支付接口返回502", "订单服务返回503")  → 应该靠近

负对（不同类）：
  ("支付接口返回502", "磁盘使用率95%")   → 应该远离
```

N=500 条可以组合出约 125,000 个训练对，这是 SetFit 在小样本下有效的核心原因——标注数据少，但训练信号多。

然后把每条文本喂进预训练 encoder（比如 `bge-small-zh`），得到向量，计算 loss：

```python
# 伪代码，说明原理
vec_a = encoder("支付接口返回502")   # shape: [768]
vec_b = encoder("订单服务返回503")   # shape: [768]
vec_c = encoder("磁盘使用率95%")     # shape: [768]

sim_pos = cosine(vec_a, vec_b)  # 希望接近 1
sim_neg = cosine(vec_a, vec_c)  # 希望接近 -1 或 0

loss = max(0, margin - sim_pos + sim_neg)
```

梯度从 loss 反传回 encoder，encoder 的权重更新，下一次 forward 时同类工单的向量会更靠近。反复迭代几百步，encoder 就从"通用中文理解"变成了"运维工单语义感知"。

---

### 第二阶段：训练分类头

encoder 调好之后冻结，重新 encode 所有 500 条，得到 500 个向量：

```
[0.12, -0.43, 0.87, ...]  → label: "接口报错5xx"
[0.09, -0.41, 0.85, ...]  → label: "接口报错5xx"
[-0.55, 0.23, -0.12, ...] → label: "磁盘空间不足"
...
```

用这 500 个向量训练一个 Logistic Regression。LR 要学的东西很简单：找一组权重矩阵 W（shape: `[768, num_classes]`），使得 `softmax(W · vec)` 输出的概率分布和真实 label 最吻合。

```python
# 训练完之后，推理就是这两步
vec = encoder("生产环境 nginx 返回 504")    # 768维向量
probs = softmax(W @ vec)                    # 每个类别的概率
# probs: {"接口报错5xx": 0.91, "网络超时": 0.06, "磁盘不足": 0.03}

label = argmax(probs)       # "接口报错5xx"
confidence = max(probs)     # 0.91
```

---

### 训练过程中实际发生了什么

本质上是两次参数优化：

第一次优化 encoder 的权重（~22M 参数，bge-small），目标是让向量空间的几何结构对齐你的类别边界。

第二次优化 W 矩阵（`768 × num_classes`，几千个参数），目标是在已经对齐好的向量空间里找到最优的切割超平面。

两次优化解耦，各自专注自己的目标，这是 SetFit 比 full fine-tune 在小样本下更稳的根本原因。full fine-tune 把这两件事混在一起用 110M 参数同时学，500 条数据根本撑不住。
