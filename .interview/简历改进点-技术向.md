## 二、弱动词分级：哪些能升，哪些保持诚实

不是所有「参与」都改「主导」——会在追问时崩。分层：

- **OpsPilot（个人项目）**：放心用「设计/实现/自建/主导」。
- **开发套件 / 应用生成平台（公司团队项目）**：保留诚实，把「参与规划」收敛到真正主导的子模块。
  - 开发套件本就「主导前端研发与 UI 自动化测试方向」，可加实现细节：
  - ✅ 主导前端验证能力域，拆为 **8 个原子节点**（范围识别→用例生成→Playwright 执行→证据采集→**五分类失败归因**→缺陷流转→复测），用浏览器截图/Console/Network/Trace 做 evidence pack 替代 LLM 口头验收

## 三、技能段：从「产品词云」改成「工程关键词」

现状 AI/Agent 行全是名词堆叠，缺工程深度词。改为面试官会搜的词（作战计划链路一~八均可扛）：

> **Agent 工程**：LangGraph（Pregel/BSP/条件路由/环形图）、checkpoint 持久化与续跑、Human-in-the-loop、evidence-driven 门禁、扇出并发（asyncio.gather）
> **后端可靠性**：FastAPI async / Event Loop、SSE 断线续传、Task Lease、乐观锁防 Lost Update、Tool Gateway 幂等（幂等键设计）、Saga 补偿
> **RAG**：混合检索（BM25+向量 RRF）、Rerank（cross-encoder）、Chunking、意图识别

作用：埋钩子，引导面试官问你的强项。

现 5 段中 2 段在说「业界 evaluation harness / Agent Skills 渐进式披露 / MCP 标准化」——产品 BD 话术，工程岗看了觉得空。留 1 句点题，腾出空间写技术判断（用作战计划开场内核）：

## 五、量化措辞：去掉「预计」

OpsPilot「问题定位效率**预计**较人工提升约 30%-50%」——「预计」一出现可信度归零，且本就是模拟演练。

- 改法：删百分比，只留「在模拟工单演练中验证闭环价值」；或诚实标注「模拟演练数据」。
- 真实可信数字（**2.2 天→4 小时**、**30+ 原型**）前置加粗。
